Для чего нужны методологии?

Проблемы CSS

  • Маштабируемость
  • Колличество кода
  • Поддержка кода
  • Страктура (стилистика)

Требования к верстке

  • Изменения кода не должно ломать старый код
  • Разработчик должен понимать свой код
  • Уменьшение повторяющегося кода
  • Команда должна говорить на "одном языке"

Критерии оценки методологии*

  • Понимаемость кода - Вернувшись к коду через год должно быть все понятно и не только вам
  • Масштабируемость - на сколько безболезненно можно создавать новые елементы
  • Колличество кода - Влияние на оптимизацию, исключение повторения кода
  • Поддержка - Добаботка Legacy кода
  • Порог вхождения - Время на изучение методологии, реализация ее, и шаринга в команде

*По мнению "редакции"

OOCSS Object-Oriented CSS

Концепция:

В этот подход заложены две основные идеи:

  • Разделение структуры и оформления
  • Разделение контейнера и контента (содержимого)

OOCSS Object-Oriented CSS

OOCSS Object-Oriented CSS

Плюсы:

  • Меньшение количества кода за счет повторного его использования
    (принцип DRY).

  • Подход предлагает не конкретные правила, а абстрактные рекомендации

OOCSS Object-Oriented CSS

Хороший подход если у вас на календаре еще 2007 год.

BEM (БЭМ) Block Element Modifier

Концепция:

компонентный подход к веб-разработке. В его основе лежит принцип разделения интерфейса на независимые блоки.

Он позволяет легко и быстро разрабатывать интерфейсы любой сложности и повторно использовать существующий код

BEM (БЭМ) Block Element Modifier

Разделение всех возможных компонентов на 3 сущности

  • Блок
  • Элемент
  • Модификатор

Блок (BEM)

Логически и функционально независимый компонент страницы. Блок полностью самодостаточен: у него может быть свое поведение, шаблоны, стили, документация и не только. Блоки могут использоваться в любом месте страницы, повторно, даже в другом проекте.

Одни блоки можно вкладывать в другие, компоновать, использовать для создания более сложных блоков.

Элемент (BEM)

Часть блока, которая не может использоваться в отрыве от него и имеет смысл только в рамках своего родителя. Элементы могут быть обязательными и опциональными.

Модификатор (BEM)

Свойство блока или элемента, которое меняет их внешний вид, состояние или поведение. У блока/элемента может быть несколько разных модификаторов одновременно.

BEM (БЭМ) Block Element Modifier

Правила формирования имени БЭМ-сущности

  • Каждая БЭМ-сущность должна иметь свой класс.
    (Нюанс а том что б не переопределять теги)
  • CSS-свойства для блоков, элементов и модификаторов описываются только
    через классы.
  • Для разделения слов в именах используется дефис (-).
  • Элемент отделяется от блока двумя подчеркиваниями (__).
    Модификатор — одним (_).
  • Имена БЭМ-сущностей записываются с помощью цифр и латинских букв в нижнем регистре.

BEM (БЭМ) Block Element Modifier

Плюсы:

  • Относительная независимость блоков
  • Избавление от каскадности
  • Избавление от копирования

Минусы:

  • Ужасные css селекторы
  • Черезмерна сложная разработка и поддержка для небольших проектов
  • Жесткая привязка к названиям стилей
  • Инструментарий БЕМ избыточный

BEM (БЭМ) Block Element Modifier

Мнение редакции:

БЭМ придуман компанией Яндекс и данная методология создана для покрытия их проблем.

Если ваш процесс работы отличается от яндекса, можете забыть про бэм, иначе будете плодить говнокод

SMACSS Scalable, Modular Architecture CSS

Концепция:

SMACSS расшифровывается как «масштабируемая и модульная архитектура для CSS» (Scalable and Modular Architecture for CSS).

Подход предлагает разделить стили на 5 частей
(в порядке включения их в документ)

SMACSS Scalable, Modular Architecture CSS

Компоновка модулей

  • Base - Стили ко всем семантическим или базовым стилям CSS без использования классов
  • Layout - Основные компоненты #header, #article, #footer
  • Module - Многоразовым модульным элементом в дизайне
  • State - Состояния любых компонентов, используется с префиксом is-
    (is-active, is-inactive, is-expanded, is-hidden)
  • Theme - Используется если у вас есть различные шаблоны тем

SMACSS Scalable, Modular Architecture CSS

Компоновка модулей

SMACSS Scalable, Modular Architecture CSS

Плюсы:

  • Подход предлагает правильные рекомендации для модульного и поддерживаемого CSS-кода, избегая при этом излишних предписаний
  • Система достаточно гибкая, чтобы работать с крупными и маленькими проектами.
  • Легкий в освоении
  • Хорошая модульность

Минусы:

  • Семантический подход
  • Ограниченная вложенность

SMACSS Scalable, Modular Architecture CSS

Мнение редакции:

Хорошая методология, очень логическое разделение компонентов сайта, но такой подход поошряет использование семантических элементов и обращение к id

RSCSS Reasonable System for CSS

Разумная система для структуры стилей CSS.

Концепция:

Реализация модульности за счет вложенности елементов в модуле

RSCSS Reasonable System for CSS

 



Примерно так будет выглядеть код модуля при использовании такого подхода

RSCSS Reasonable System for CSS

Компоновка модулей

  • Component - Независимые блоки
  • Element - Составные части модулей, не существую вне модуля
  • Layout - Слои приложения. Расположение областей
  • Variant - Модификаторы или состояния
  • Helpers - Вспомогательные элементы

RSCSS Reasonable System for CSS

Плюсы:

  • Среднее между БЭМ и SMACSS
  • Более лаконичное написание кода
  • Еще более назависимые стили
  • Легкост в написании
  • Отлично подходит для небольших и средних проектов

Минусы:

  • Каскадность. При большой вложенности создает больше проблем
  • DRY - это не про этот подход

RSCSS Reasonable System for CSS

Мнение редакции:

Один из самых легких в освоении, и гибкий в написании и поддержке подход.

В добавок стиль написания довольно красивый если не будет большой каскадности

Итоги:

Как видите, подходов к организации вашего CSS очень много, на любой вкус и цвет. Но у всех в той или иной степени есть свои плюсы и минусы, и нет универсального решения.

Важно знать и иметь общее представление о существующих методологиях, для того что б вы могли понимать для какого проекта, какой подход больше подойдет

Но главное это то что правильное и уместное использование методологий сделает вашу работу и работу вашей команды с CSS намного приятнее
